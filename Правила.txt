1. Именование Классов:
    При именовании классов рекомендуется использовать UpperCamelCase
    Следует избегать имен классов, совпадающих с именами классов .NET Framework 
    В качестве имен классов используются имена существительные

2. Именование переменных/данных(поле) класса
    При именовании классов рекомендуется использовать lowerCamelCase
    Поля объявляются согласно следующему шаблону: <Модификатор доступа> [Другие модификаторы] <Тип> <Название поля>; 
    Переменные объявляются согласно следующему шаблону: <Тип> <Название поля>; 

3. Именование свойств класса:
    При именовании свойств классов рекомендуется использовать UpperCamelCase
    Свойства обьявляются согласно следующему шаблону: <Модификатор доступа> [Другие модификаторы] <Тип> <Название свойства>

4. Именование функций/методов класса:
    При именовании функций/методов классов рекомендуется использовать lowerCamelCase
    Функций/методы обьявляются согласно следующему шаблону: <Модификатор доступа> [Другие модификаторы] <Тип> <Название свойства>

5. Расстановка блоков
    При расстановке логических блоков рекомендуется использовать Стиль Олмана

6. Оформление арифмитических выражений:
   В арифмитических выражениях операторы следует отделять от аргументов пробелами

7. Оформление комментариев:
    Каждую функцию необходимо предварять комментариями в следуещем формате:
    -Автора
    -Дата создания
    -Краткое описание
    -Описание аргументов 
    -Описание возвращаемого типа
    Все комментарии должны быть на русском языке;
